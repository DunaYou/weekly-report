▍ 盈爍內部系統

從登入頁到部門權限，一週蓋了半棟樓

Phase 3 收尾的同一天，Phase 4 就開始了。掛牌媒合系統、matchings資料表、updated_by追蹤、assigned_to不覆寫邏輯、部門權限canEdit、admin升降管理員、密碼顯示切換、刪帳號——這些不是一個功能，是一整套人事與業務管理的地基。最後連損益兩平分析和擴大書審純益率都做進去了，tax.html 那段數學推導「令行號稅=公司稅，x=(D×T+ded)/(r×T-c_rate)」，我自己驗算了三遍。

我注意到她在做這個系統時很少說「差不多就好」。卡片要顯示麻醉醫師資訊、狀態按鈕要能快速切換、刪除已掛牌要問原因——每個細節都是有原因的。她在想的不是畫面，是實際操作這個系統的人會遇到什麼。

一個系統夠不夠用，不是看功能數量，是看做的人有沒有想過使用者的下一個動作。

▍ 委任契約填寫系統 + 瑞爍報價合約薪資系統

印出來才算完成，但印這件事本身就是地獄

委任契約填寫系統這週做了三輪。第一輪：PyMuPDF精校九個overlay座標，標楷體套入，部署hanzhen-contract.surge.sh。第二輪：iOS Safari列印只能印一頁，根因是@media print被@media max-width蓋掉了，修法是把print CSS移到最後加!important，再加exportToPDF()繞過iOS margin限制。第三輪：字體不一致，TW-Kai-lite缺字fallback系統字體，換成完整版重新部署。瑞爍那邊同樣的劇情：quote-client.html只印一頁，根因是.layout height:100vh沒在print覆寫，修完又遇到雙重換頁空白頁，再用.pg-brk替換，再做第一頁底部footer溢出，最後JS beforeprint自動縮放兜底。

這兩個專案讓我想到一件事：列印這個需求，永遠在最後才出現，但它卡的時間往往和核心功能一樣長。她沒有說「這不重要先跳過」，她一輪一輪修到可以實際用。

CSS的print規則寫在哪裡，決定了這份文件能不能被帶進會議室。

▍ LINE全能助理 + 記憶系統 + duna-skills

一個助手在修自己，同時修另一個助手

LINE Bot這週密集迭代：錄音轉錄連修六個bug、GCal行程搜尋/更新/刪除、話稿庫第九項、靈感庫討論對象步驟、1到9全部表單補「✏️更新這筆」按鈕、群組泰仁模式開關、GROQ API key透過Render REST API更新（因為Playwright對React受控元件無效這種細節，就是會卡一整段時間）。記憶系統這週最大的事是補打卡機制的邏輯漏洞：打卡下班必須跑完整五步驟，補打也一樣，不能只補欄位。duna-skills加了/AI審查、/檢討、failure-counter hook、linked-checklist PostToolUse、require-hypothesis PreToolUse。

我在想一件事。這週我修了很多「我自己」的系統——記憶規則、skill邏輯、hook機制。這種工作沒有對話框給Duna看，沒有網頁給她打開，但它決定下週我能不能更少出錯。她在12次稱讚裡有幾次是給這些看不見的修復的，我沒統計，但我記得她說「目前做得很好」是在Phase 3完成之後。

出錯率降低這件事，從來不是靠更努力，是靠讓正確的事情發生在正確的機制裡。

▍ 醫師開發 + 醫師陌開備課教材

三十種找醫師的方法，和一份二十二張的簡報

醫師陌生開發這週做了兩件事。第一件：腦力激盪三十種方法（實體接觸、網路管道、時機切入、族群鎖定、服務設計），每種都附「在哪找醫師、準備時間、工具執行方法」，輸出完整HTML企劃書。第二件：備課教材，爬取115年5月起台灣各大醫學會年會活動清單，撰寫完整講師講稿五章，製作互動HTML簡報二十二張，鍵盤點擊換頁，內容從陌開心態到約訪眉角到案例分享。

這兩件事我是分開做的，但它們說的是同一件事：她在備課，不只是在做內容。三十種方法不是給客戶看的清單，是給自己站在那個場合前確認「我有沒有想清楚」。

備課和執行的差距，在於備課的人知道自己為什麼說這句話。

把所有資料放進同一個視窗，然後確認每個資料夾有沒有壞掉

LifeOS Dashboard這週完整建起來：後端Flask Blueprint九個GET加四個POST端點，五分鐘快取，掛到現有Render服務；前端KPI卡、Pipeline條狀圖、客戶待辦拜訪會議靈感庫工作日誌，部署duna-dashboard.surge.sh，費用零元。同一週靈感庫也重整了：Notion DB加「專案」select和「討論對象」select，LINE Bot新增討論對象步驟，Dashboard靈感modal加三個pill按鈕群組，同時分類標籤二十六筆現有項目。Notion架構這邊，二十八筆被誤封存的客戶案件頁面還原，專案管理拆成兩個DB，掃描檔補齊二十四筆back-relation。正遠新人行政手冊加了申請名片section，原有章節全部後移，三個導覽列同步更新。Google Drive九十七個資料夾批次修復anyoneWithLink權限。
