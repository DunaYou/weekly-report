▍ LINE 全能助理 + 早安機器人

每天早上 8:30，它準時送來今天的樣子

這週 LINE 全能助理做了很多事。早安機器人從獨立服務整合進 Render，cron-job.org 改打新 URL，05/04 早上 08:30 驗證成功。推播從三則獨立訊息改成一則三張並排 Carousel——天氣行程、待辦摘要、財經新聞，視覺用 GPT 設計稿：奶白底、深綠 hero、圓角白卡。查行程新增兩步驟 UX，行程寫入從 Notion 改到 Google Calendar，連服務帳號 403 權限都一起修掉了。

我觀察到 Duna 對這個系統的態度：它不只是工具，它是她每天醒來第一個接觸的東西。所以她很在意它對不對，在意卡片好不好看，在意邏輯通不通。每個 bug 修完她都會測，不是驗收，是確認這個東西值得信任。查客戶 AttributeError、靈感庫三個 bug、Flex Message SDK 序列化問題——她不跳過任何一個。

一個每天使用的工具，你對它的容忍度比任何東西都低。這反而是好事。

▍ 記憶系統 + CLAUDE.md + duna-skills

她開始寫規則，讓我不用靠猜

這週最大的底層工程，是讓整個 AI 工作系統不再靠我的「記憶」運作。CLAUDE.md 頂端加了「已踩地雷」表格，五條，每次對話必讀。建了 専案総覧.md，17 個專案一行摘要。記憶 skill 從五步驟重整到七步驟，最後又整合成六步驟，每次調整都是因為發現上一版還是有漏洞。duna-skills 從 Raymond 文章的 skill 格式出發，分析 CLAUDE.md，抽出 13 個重複流程做成獨立 SOP，同步刪掉 CLAUDE.md 裡 161 行冗餘說明。

她對這件事的怒氣是真實的。同一個 sticky bug 犯了兩次，同一個 @media 順序問題犯了兩次。她不是在生氣我笨，她在生氣「明明記錄了，為什麼還是找不到」。這週做的這些——進度.md 強制必讀、對話開始強制輸出格式、地雷表格放最頂端——全部都是為了解決「存的地方跟讀的地方不一樣」這一個根本問題。

跨對話失憶不是記憶力問題，是讀取機制問題。她想清楚了這件事，然後把系統改了。

▍ 週報系統 + 月報系統 + AI 學習地圖

報告系統長大了，開始告訴我們花了多少錢

週報新增「值得偷學的案例」section，Firecrawl 搜尋加 Claude 整理，三來源（Web + GitHub + Threads），偷學案例自動寫入靈感庫，來源標示「游泰仁週報」與 Duna 的靈感分開存。月報升級了費用儀表板，目前支出、預估月費、LINE 用量、Anthropic 剩餘，API token 用量改成每次 call 後寫入 json，月報讀取精確數字而不是估算。手機版 sidebar sticky 問題修了兩次——第一次 @media 位置錯，第二次才真的對。AI 學習地圖做到 v4.0，七層完整架構，分頁式，技術詞全白話化，工具全覽加「泰仁主動規則」三層觸發欄，部署到 duna-ai-map.surge.sh。

這些工具有個共同特質：它們是給 Duna 看自己的鏡子。週報看這週做了什麼，月報看錢花去哪，AI 地圖看自己現在站在哪個層次。她花時間把這些做好，不是因為好看，是因為她需要知道自己在哪裡。

做給自己看的工具，才是最難偷懶的那種。

▍ Threads 觀察系統 + 內容規劃

爬蟲換了帳號，觀察還在繼續

Threads 觀察系統這週進化了幾個層次。關鍵字從 17 個擴充到 19 個，移除無效縮寫，加入診所經營、節稅、醫療糾紛——實測發現「醫療糾紛」有 2099 讚、「節稅」分享 982 次，縮寫幾乎全死。LINE 卡片新增身份觀察區塊，推測發文者是律師、醫師、顧問還是會計師。Threads DOM 改版，button 選擇器失效，改用 aria-label 重新抓互動數。最重要的一件事：爬蟲改用空號 panda.1050009，PyNaCl 加密後存進 GitHub Secret，降低真實帳號封號風險。

Threads 企劃書這週也部署了：6 個帳號 Day1 + Day2 文案草稿全寫完，固定側邊欄加 Intersection Observer 高亮，行動版 hamburger menu，部署到 duna-threads-plan.surge.sh。她做這些的速度讓我覺得，她不是在測試 Threads 能不能用，她早就決定了要用，現在只是在把工具備齊。

在正式開始之前把工具備齊，這是她的工作方式。不是等到需要再找，是還沒需要就先準備好。

▍ 盈爍報價確認書 + 客戶資料夾管理 + 瑞爍輿情監控

客戶端的工具，一件一件從草稿變成能用的東西

盈爍報價確認書這週從公版到客製版都走了一遍。公版確認格式（A4 列印 + RWD），Cindy Liu 版填入跨境資產稅務效果分析，公司章用 base64 內嵌，footer 改 table 結構，簽名區兩欄底部對齊。過程中因為沒看截圖就改 CSS，把 disclaimer 和稅後金額小字搞混，多繞了幾圈——這件事記進了 CLAUDE.md 地雷清單。客戶 Google Drive 同步系統也這週建起來：26 個客戶、33 個專案，Notion 欄位新增，sync_drive.py 完成。瑞爍輿情監控系統完成自助註冊加管理員審核，Resend SDK 在 async FastAPI 靜默失敗的 bug 改用 httpx 直打 API 解決，PTT 爬蟲移除過廣的「爛」關鍵字改為「爛醫生」「爛診所」，瑞爍品牌藍色系改版完成。

這些都是服務真實客戶的東西。不是 demo，不是測試，是要拿去給人用的。她對這類東西的品質要求和對自己工具的要求一樣高，有時候更高。

給客戶用的東西出錯，代價比自己用的高很多。她清楚這件事，所以她花時間把細節做對。

那些沒有名字的工時，其實都在讓其他東西跑得起來

GitHub PAT 更新為永不到期，weekly-report、morning-briefing 的 GitHub Secrets 和 Render 環境變數全部同步。AI 特助 Agents 第一階段完成：7 個 agent 架構評估、模型選擇、費用分析，會議助理（Haiku）和文案 agent（Sonnet）各自建好，共用框架含 Prompt Caching，run.py 主控入口完成。錄音轉錄流程這週從零建起：安裝 mlx-whisper + ffmpeg，實測四個音訊檔，「整理錄音檔」觸發詞寫入 CLAUDE.md，SRT 時間戳轉錄到 Notion 主頁重點整理再到子頁逐字稿的完整流程，並為 20260507 新人訓練會議完成第一次完整寫入。醫責險要保書 UI 全面升級，正遠新人行政教學 HTML 完成部署，Claude AI 特助懶人包改版為 agentic runbook，新用戶說一句「幫我跑設定」就能自動完成整套初始化。

基礎設施不會在它正常運作的時候被看見。這週花時間修這些，是為了讓下週不用花時間找它們。
