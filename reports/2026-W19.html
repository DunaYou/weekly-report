第 19 週。18 個 sessions，14 次稱讚。她沒有說「辛苦了」，但她說了 14 次「不錯」。這週的工作不是一件事，是很多件事同時在長。

▍ Threads × IG 醫師帳號矩陣

六個帳號的第一天，文案和圖都不能少

這週花最多時間的專案，是那份給醫師客戶看的 Threads × IG 帳號矩陣企劃書。六個帳號、Day1 加 Day2 文案全部寫完，嵌入可展開的帳號矩陣區塊，加了固定側邊欄和 Intersection Observer 高亮，行動版做了 hamburger menu，最後部署到 duna-threads-plan.surge.sh。中間還處理過一次圖片在手機上跑版的問題——整資料夾重新部署、圖片全換英文檔名、橫向滑動 CSS 補上去。

她對這份報告書的細節要求是真實的。手機版跑版這件事，本來可以說「桌機看沒問題就好」，但她沒有。這讓我想到，她在意的不是「做完」，而是「對方拿到之後看起來像什麼樣子」。這種在意，是很難教的。

一份企劃書能不能被信任，有時候取決於手機版的字有沒有破版。
📅 本週進度
05/06修復手機圖片跑版問題，重新部署 surge.sh，同時完成六帳號 Day1+Day2 文案草稿全寫完
05/07用 MCP Playwright 實測 12 個 Threads 關鍵字觸及數，發現縮寫全死，高流量詞是「醫療糾紛」和「節稅」
⏱ 累計工時累計 15.1h

💡 關鍵字調查可以在企劃書初稿前就做，這樣 Day1/Day2 文案可以直接帶入高流量詞，不必事後回頭補。手機版跑版問題可以在本地端先用 DevTools 模擬，減少部署後才發現的來回次數。

▍ LINE 全能助理 + 早安機器人

早安機器人終於每天 8:30 準時打卡上班

早安機器人這週正式整合進 Render.com。cron-job.org 改打 Render URL，08:30 驗證成功。同一天，新增每月外部服務用量報告，設定每月 1 日 09:00 自動推播到 LINE。這個月報功能是之前討論過的，現在真的跑起來了。

LINE 全能助理這週也加了不少東西：修了靈感庫三個 bug、新增「8. 新增專案」選項解決 Notion multiple data sources DB 的 API 版本問題、做了查行程的兩步驟 UX、修了 FlexMessage SDK Pydantic 序列化的問題——改用 `_reply_flex_raw` 直接 POST LINE API 才通。她在這個專案上的耐心讓我印象深刻，每次 bug 修完，下一個功能就接著進來，沒有停。

一個工具能不能被每天用，往往不是功能夠不夠多，而是 8:30 那則訊息有沒有準時出現。
📅 本週進度
05/04早安機器人整合到 Render，cron-job.org 設定完成，早上驗證成功，同步更新 CLAUDE.md 與 memory
05/04修復靈感庫 3 個 Bug，整合 morning.py 模組，程式碼推上 GitHub
05/05LINE Bot 新增「新增專案」功能，用 API 2025-09-03 解決 multiple data sources DB 問題
05/07修復 FlexMessage 序列化問題，實作查行程兩步驟 UX，四個檔案語法確認後推送
⏱ 累計工時累計 6.5h（LINE 全能助理）+ 1h（LINE 早安機器人）

💡 FlexMessage 的 Pydantic 序列化問題是老問題，可以在共用工具函式層統一處理，不必每次遇到才修。查行程的兩步驟 UX 可以考慮加一個快速指令（例如「今天行程」直接跳過問答流程），減少步驟。

▍ AI特助Agents + 會議錄音升級

七個 Agent 的骨架，這週立起來了

AI 特助 Agents 第一階段這週完成。架構評估做完了——七個 agent、模型選擇、費用分析全部確認。共用框架、會議助理 agent（Haiku）、文案 agent（Sonnet）、run.py 主控入口全部建立。Prompt Caching 也已經套進去。LINE /remind endpoint 部署，5/17 費用確認提醒設定好了。

同一天，BlackHole 2ch 虛擬音效裝好了，等重開機後設定 Audio MIDI Setup，讓 LINE 電話雙邊都錄得到。這兩件事放在同一天做，是因為它們底下在解同一個問題：會議的內容要能被處理，先要能被錄到，再要能被分析。她把這兩件事放在一起，說明她看的不是功能，是流程。

系統的第一階段，通常是在確認「這個方向對不對」，而不是「功能夠不夠完整」。
📅 本週進度
05/04完成 AI 特助 Agents 架構評估、共用框架、會議助理與文案 agent，安裝 BlackHole 2ch 虛擬音效
05/04第一階段收尾：建立 run.py 主控入口，部署 /remind endpoint，設定費用確認提醒
⏱ 累計工時累計 5h（AI特助Agents）+ 2h（會議錄音升級）

💡 七個 agent 的費用分析做完了，但實際費用只有跑過測試，建議在 5/17 確認後補一次真實用量對照，讓預估和實際之間有一個基準。BlackHole 設定等重開機後完成，不要拖過下週，不然錄音流程會繼續卡在半完成狀態。

▍ 醫責險要保書填寫系統

UI 改了一輪，流程圖也跟著換新衣服

醫責險這週做的是 UI 全面升級。fbmmiclient 把填寫說明改成 modal、診所備注改 inline、賠償和爭議欄位加了提醒文字。流程圖那邊，字體放大、QR code 縮小橫排、Step 1-4 和 Step 5-8 改兩欄排版。fbmmiensure 那邊，移除無自負額版標題、匯入按鈕改名為「掃描或貼上授權碼」、操作說明換成流程教學圖。Notion 投保流程頁面也同步更新：圖片換成 v2、Step 2 加了填寫說明提示、URL 改成可點擊超連結、link_mention 快取問題修正。

這一輪改的不是功能，是使用者拿到之後的第一眼感受。modal、inline、提醒文字——每一個決定都是在想「如果是第一次填這份表的人，他會在哪裡卡住」。這種視角，是做產品的人才有的，不是做系統的人。

表單的 UI 改一次，客服問題少一半。這不是誇張。
📅 本週進度
05/04fbmmiclient、flow_chart、fbmmiensure 三個頁面 UI 全面升級，Notion 投保流程頁面同步更新至 v2
⏱ 累計工時累計 4.5h

💡 流程圖兩欄排版在手機上可能需要確認一次，Step 文字在小螢幕是否還清楚。Notion link_mention 快取問題這次是手動修，可以評估是否需要寫一個自動重整的小腳本，避免下次更新後又要手動處理。

▍ Threads 帳號矩陣 + 貼圖 + 生態觀察系統

做圖流程建起來了，觀察系統也長了新眼睛

Threads 這條線這週同時跑了三件事。第一件：做圖流程從零建立——prompt → Playwright → ChatGPT 生圖，測試了 12 張教學卡，盈燦 6/7 和 6/14 的講座 DM 也一起生出來。第二件：Threads × IG 企劃報告書完成部署，封面加上 Duna×WYBX×游泰仁，Section 09 待確認內容標注好，整份報告書可以交出去了。第三件：生態觀察系統升級，關鍵字從 17 個擴到 19 個，LINE 卡片新增身份觀察區塊，可以推測發文者是律師、醫師、顧問還是會計師，同時評估流量加成。

她在 Threads 這條線上同時做了內容生產工具、對外提案文件、和觀察情報系統三件事。這三件事表面上都叫「Threads」，但底下的邏輯完全不同——一個是流程、一個是說服、一個是情報。能同時在三個維度推進，不容易。

觀察系統能看到別人看不到的東西，才值得花時間建。
📅 本週進度
05/04建立 Threads 做圖流程，測試 12 張教學卡 prompt，生成盈燦講座 DM
05/06Threads × IG 企劃報告書完成，六帳號文案嵌入，封面與側邊欄完成，部署到 surge.sh
05/07關鍵字清單優化，移除無效縮寫，加入診所經營/節稅/醫療糾紛，生態觀察系統新增身份分析卡片
⏱ 累計工時累計 5.5h（Threads 帳號矩陣）+ 1.8h（Threads 貼圖）+ 1.8h（盈燦講座 DM）+ 1.5h（Threads 生態觀察系統）

💡 做圖流程目前是手動觸發，如果教學卡的需求量增加，可以考慮包成一個小腳本讓 Duna 自己執行。生態觀察系統的身份推測目前是規則型，下一步可以考慮用少量 prompt 讓模型直接判斷，準確率應該會更高。

▍ 正遠新人行政教學 + 客戶資料夾管理 + 系統維護

新人教學上線，Google Drive 客戶資料夾也補齊了

正遠新人行政教學這週完成部署。13 個章節、綠色設計、左側目錄、手機支援，部署到 ensrueright-newbie.surge.sh。後來又做了細節修正：送件清單 PDF 改 Google Drive 直連、正遠文件改編號清單、產險合格證號連結更新。文件做好不是終點，連結對、格式對，新人才真的能用。

客戶資料夾管理這週建了 Google Drive 同步系統，26 個客戶、33 個專案，Notion 欄位新增，sync_drive.py 完成。另外，GitHub PAT 這週 regenerate 成永不到期，weekly-report、morning-briefing 的 GitHub Secrets 和 Render 環境變數全部同步更新，monthly_report.py 的到期日改成 2099-01-01。這些事情沒有做完就不知道有多脆弱，做完才知道之前有多多條線卡在快到期的 token 上。

Notion 專案管理補建了五筆部署條目，確立類型欄位四分法：開發、公司營運、客戶、個人。往後有部署的專案自動建 Notion 條目，這個規則從這週開始生效。

基礎設施的問題，不痛不癢，直到它痛了才知道有多嚴重。PAT 永不到期這件事，就是在問題發生之前把它解決。
📅 本週進度
05/05正遠新人行政教學 HTML 完成，部署到 Surge.sh，13 章節全部到位
05/06GitHub PAT regenerate 成永不到期，全平台同步更新 Secrets 與環境變數
05/06正遠細節修正完畢，送件清單改直連，文件清單改編號格式
05/07Notion 專案管理補建五筆條目，確立類型四分法，寫入自動建條目規則
05/08建置 Google Drive 客戶資料夾同步系統，26 客戶 33 專案，sync_drive.py 完成
⏱ 累計工時累計 2.5h（正遠新人行政教學）+ 1.5h（客戶資料夾管理）+ 0.5h（Notion 專案管理）+ 0.5h（記憶系統）

💡 sync_drive.py 目前是手動跑，可以考慮接進 cron 定期執行，確保 Notion 和 Drive 之間的資料不會因為手動忘記同步而跑掉。正遠新人教學的連結日後若有變動，需要一個更新機制，不然 surge.sh 上的靜態頁面會繼續指向舊連結。
